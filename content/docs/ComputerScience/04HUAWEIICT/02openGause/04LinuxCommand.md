**摘要：** 无论您是 Linux 新手还是希望巩固基础的开发者，一份全面、实用的命令手册都是不可或缺的。本教程超越了基础的 `ls` 和 `cd`，深入探讨了文件处理、进程管理、系统监控、用户权限、网络诊断等多个维度的核心命令，并特别讲解了管道 (`|`) 和重定向 (`>`) 这两个提升效率的利器。最后，我们还提供了一份详尽的常用命令参数附录，旨在成为您值得收藏的 Linux 命令实战宝典。

## 一、Linux 命令行的灵魂：管道与重定向

理解这两个概念，能让您将简单的命令组合成强大的工具链。

*   **重定向 (`>` 和 `>>`)**：改变命令输出的目标。
    *   `>`：**覆盖写入**。将命令的输出写入一个文件，如果文件已存在，其内容将被覆盖。
    *   `>>`：**追加写入**。将命令的输出添加到文件的末尾，而不会破坏原有内容。
    ```bash
    # 将 /etc 目录的列表覆盖写入到 etc_content.txt
    ls -l /etc > etc_content.txt

    # 将当前日期追加到上面的文件末尾
    date >> etc_content.txt
    ```

*   **管道 (`|`)**：将一个命令的输出，作为另一个命令的输入。这是 Linux 哲学“做一件事并把它做好”的体现。
    ```bash
    # 查找所有正在运行的 ssh 进程
    # ps aux 列出所有进程，其输出通过管道传给 grep 进行过滤
    ps aux | grep ssh
    ```

## 二、文件与目录管理

这是最基础也最重要的部分，涵盖了创建、查找、移动、打包等日常操作。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`pwd`** | 显示当前工作目录的绝对路径 (Print Working Directory)。 | `pwd` |
| **`ls`** | 列出目录内容 (List)。 | `ls -alh` (显示所有文件、长格式、人类可读) |
| **`cd`** | 切换目录 (Change Directory)。 | `cd /var/log` (绝对路径), `cd ..` (上级目录), `cd ~` (家目录) |
| **`mkdir`** | 创建新目录 (Make Directory)。 | `mkdir -p /data/app/logs` (-p 递归创建) |
| **`touch`** | 创建空文件或更新文件时间戳。 | `touch new_file.log` |
| **`cp`** | 复制文件或目录 (Copy)。 | `cp -r /source/dir /target/dir` (-r 递归复制目录) |
| **`mv`** | 移动或重命名文件/目录 (Move)。 | `mv old_name.txt new_name.txt` (重命名), `mv file.txt /tmp/` (移动) |
| **`rm`** | 删除文件或目录 (Remove)。{{< admonition danger >}}**极其危险，无回收站！**{{< /admonition >}} | `rm file.txt` (删除文件), `rm -rf /path/to/dir` (强制递归删除目录) |
| **`ln`** | 创建链接文件 (Link)。 | `ln -s /usr/bin/python3 /usr/bin/python` (-s 创建软链接) |
| **`find`** | 在指定目录中查找文件。 | `find /opt -name "*.xml"` (在 /opt 目录下按名称查找 xml 文件) |
| **`tar`** | 打包和解包文件 (Tape Archive)。常与压缩结合使用。 | `tar -czvf archive.tar.gz /path/to/dir` (打包压缩)<br>`tar -xzvf archive.tar.gz` (解包解压) |

## 三、文件内容查看与处理

高效地查看、搜索和编辑文件内容是排查问题的关键。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`cat`** | 查看整个文件内容 (Concatenate)。 | `cat /etc/profile` |
| **`head`** | 查看文件开头部分，默认10行。 | `head -n 20 error.log` (查看前20行) |
| **`tail`** | 查看文件结尾部分，默认10行。 | `tail -n 50 access.log` (查看后50行)<br>`tail -f app.log` (-f 实时跟踪文件更新) |
| **`grep`** | 在文本中搜索匹配的行 (Global Regular Expression Print)。 | `grep -i "error" /var/log/messages` (-i 忽略大小写) |
| **`vi/vim`** | 强大的命令行文本编辑器。 | `vim filename` (打开/创建文件)<br>模式切换：`i` (插入), `Esc` (普通), `:wq` (保存退出) |

## 四、用户与权限管理

Linux 安全的基石，规定了“谁能对什么文件做什么事”。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`chmod`** | 修改文件权限 (Change Mode)。 | `chmod 755 script.sh` (赋予脚本执行权限)<br>`chmod -R 644 /data/www` (递归修改权限) |
| **`chown`** | 修改文件所有者和所属组 (Change Owner)。 | `chown -R omm:dbgrp /opt/gaussdb` (递归修改所有者和组) |
| **`su`** | 切换用户 (Switch User)。 | `su - omm` (- 表示同时切换到该用户的环境) |
| **`sudo`** | 以其他用户（通常是root）的身份执行命令。 | `sudo yum install wget` (临时获取root权限安装软件) |

## 五、进程管理

实时监控和控制系统中运行的程序。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`ps`** | 显示当前进程的快照 (Process Status)。 | `ps aux` 或 `ps -ef` (查看系统上所有进程) |
| **`top`** | 实时动态地显示系统进程信息，类似任务管理器。 | `top` (进入后按 `q` 退出) |
| **`kill`** | 向进程发送信号，通常用于终止进程。 | `kill 12345` (优雅地终止PID为12345的进程)<br>`kill -9 12345` (强制杀死进程) |

## 六、系统信息与监控

了解服务器的健康状况。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`df`** | 查看磁盘空间使用情况 (Disk Free)。 | `df -h` (-h 以人类可读格式显示) |
| **`free`** | 查看内存使用情况。 | `free -h` (-h 以人类可读格式显示) |

## 七、网络工具与软件安装

用于网络诊断和系统软件管理。

| 命令 | 作用 | 核心示例 |
| :--- | :--- | :--- |
| **`ping`** | 测试网络连通性。 | `ping www.bing.com` |
| **`wget`** | 非交互式网络下载器。 | `wget https://.../file.tar.gz` |
| **`yum`/`apt`**| 软件包管理器。 | `sudo yum install htop -y` (CentOS/openEuler)<br>`sudo apt-get install htop -y` (Debian/Ubuntu) |

## 附录：常用命令参数详解

本章节为您详细解析常用命令的核心参数，帮助您更灵活、更精确地使用它们。

### `ls` (List)
**核心用途**：列出文件和目录。

| 参数 | 说明 | 示例 |
| :--- | :--- | :--- |
| `-l` | 使用长格式（long format）显示，包含权限、所有者、大小、修改日期等详细信息。 | `ls -l` |
| `-a` | 显示所有（all）文件，包括以 `.` 开头的隐藏文件。 | `ls -a` |
| `-h` | 配合 `-l` 使用，以人类可读（human-readable）的格式显示文件大小（如 K, M, G）。 | `ls -lh` |
| `-t` | 按修改时间（time）排序，最新的文件排在最前面。 | `ls -lt` |
| `-r` | 反转（reverse）排序顺序。 | `ls -ltr` (按修改时间从旧到新排序) |

**实战组合**：`ls -alht` 是一个非常实用的组合，它会列出所有文件（包括隐藏的），以详细、易读的格式，并按最新修改时间排序。

### `grep` (Global Regular Expression Print)
**核心用途**：在文本中进行模式搜索。

| 参数 | 说明 | 示例 |
| :--- | :--- | :--- |
| `-i` | 忽略（ignore）大小写进行匹配。 | `grep -i "Error" app.log` |
| `-v` | 反转（invert）匹配，即显示不包含匹配模式的行。 | `grep -v "DEBUG" app.log` (过滤掉调试信息) |
| `-n` | 显示匹配行及其行号（number）。 | `grep -n "keyword" file.txt` |
| `-r` / `-R` | 递归（recursive）搜索目录下的所有文件。 | `grep -r "db_password" /etc` |
| `-C <num>` | 显示匹配行的上下文（Context），即匹配行的前后各 `<num>` 行。 | `grep -C 2 "Exception" error.log` |

### `find` (Find)
**核心用途**：在文件系统中搜索文件和目录。

| 参数 | 说明 | 示例 |
| :--- | :--- | :--- |
| `-name <pattern>` | 按文件名模式查找（支持通配符 `*`, `?`）。 | `find /home -name "*.log"` |
| `-type <d/f>` | 按类型查找，`d` 为目录（directory），`f` 为普通文件（file）。 | `find /var -type d -name "log"` |
| `-mtime <N>` | 按修改时间（modification time）查找。`-N` (N天内), `+N` (N天前), `N` (正好第N天)。 | `find /tmp -mtime +7` (查找7天前修改过的文件) |
| `-size <N>` | 按文件大小查找。`+10M` (大于10MB), `-100k` (小于100KB)。 | `find / -size +1G` (查找大于1GB的文件) |
| `-exec <command> {} \;` | 对找到的每个文件执行指定的命令。`{}` 代表找到的文件名。 | `find . -type f -name "*.tmp" -exec rm -f {} \;` (查找并删除所有.tmp文件) |

### `tar` (Tape Archive)
**核心用途**：打包、压缩和解压文件。

| 参数 | 说明 | 示例 |
| :--- | :--- | :--- |
| `-c` | 创建（create）一个新的归档文件。 | `tar -cvf archive.tar /dir` |
| `-x` | 从归档文件中提取（extract）文件。 | `tar -xvf archive.tar` |
| `-f` | 指定（file）归档文件的名称。**通常是最后一个参数**。 | `tar -czf archive.tar.gz /data` |
| `-v` | 显示详细（verbose）处理过程。 | `tar -xzvf archive.tar.gz` |
| `-z` | 通过 gzip 进行压缩或解压（文件后缀通常为 `.gz`）。 | `tar -czf ...` (压缩), `tar -xzf ...` (解压) |
| `-j` | 通过 bzip2 进行压缩或解压（文件后缀通常为 `.bz2`）。 | `tar -cjf ...` (压缩), `tar -xjf ...` (解压) |
| `-t` | 列出（list）归档文件中的内容，而不提取。 | `tar -tf archive.tar.gz` |

### `chmod` & `chown`
**核心用途**：管理文件权限和所有权。

| 命令 | 参数 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `chmod` | `-R` | 递归（Recursive）地修改目录及其下所有文件和子目录的权限。 | `chmod -R 755 /var/www/html` |
| `chown` | `-R` | 递归（Recursive）地修改目录及其下所有文件和子目录的所有者/组。 | `chown -R www-data:www-data /app` |

### 文件操作 (`cp`, `mv`, `rm`)
**核心用途**：复制、移动和删除文件。

| 命令 | 参数 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `cp`, `mv`, `rm` | `-r` / `-R` | 递归（Recursive）操作，用于处理目录。 | `cp -r /source /destination` |
| `cp`, `mv`, `rm` | `-f` | 强制（force）执行，覆盖前不提示。 | `rm -f old_file.txt` |
| `cp`, `mv`, `rm` | `-i` | 交互式（interactive）操作，执行前进行提示确认。 | `rm -i *.log` |
| `cp`, `mv` | `-v` | 显示详细（verbose）过程，告知用户正在做什么。 | `mv -v file.txt /new/dir/` |
